# Runtime floors. These make the code INSTALL and RUN.
# They do NOT pin the reproducibility claim -- see requirements-frozen.txt.
#
# WHY BOTH FILES EXIST
# --------------------
# This paper claims that two machines reproduced every table to the last digit.
# A ">=" floor cannot support that claim: it is a LOWER bound, so it permits any
# future version. Someone installing from this file in 2028 may get numpy 3.x,
# get different numbers, and their requirements.txt will tell them everything is
# fine. A file that says "fine" while the results have changed is exactly the
# failure this paper is about.
#
# So: install from requirements.txt to run the code.
#     install from requirements-frozen.txt to reproduce the LOGGED NUMBERS.
